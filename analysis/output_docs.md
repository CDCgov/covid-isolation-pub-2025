# Output files, figures, and tables

>[!IMPORTANT]
>This documentation covers two topics: 1) the contents of the intermediary model output files contained in the release file `covid-isolation-outputs.zip` and 2) which script in [`analysis/scripts/products`](scripts/products) is used to produce which tables and figures and whether each product can be recreated using the publicly available model output.
## Model output files
The file structure of `covid-isolation-outputs.zip` is as follows:

```
covid-isolation-outputs/
├── development/
│   └── preprocessing_outputs/
│       └── stan_joined_inherent_rvtn_full.csv
└── production/
    ├── model_scenarios/
    │   └── compare_guidance_df.csv
    │   └── intervention_params.csv
    │   └── overall_compare_guidance_df.csv
    ├── preprocessing_outputs/
    │   └── symp_cat_weights.csv
    └── viral_kinetics_posteriors
        └── VL_symp_cat_pooled.csv
        └── VL_symp_cat_sampled.csv
```
Each file is describe below. The files are listed in the order that they are generated by the pipeline.

### `stan_joined_inherent_rvtn_full.csv`
>[!WARNING]
>Unlike the other files released in `covid-isolation-outputs.zip`, this file contains artificial synthetic data and is included only for the purpose of technical demonstration of fitting the Stan model. (See `pipeline.md` for details.) *Its contents and downstream model fits should not be used for substantive scientific purposes.* This artificial version of the file is intentionally a minimum working example; the version of the file as generated in the actual pipeline by [`01_prepare_data_for_stan.R`](scripts/01_prepare_data_for_stan.R) includes more variables about study participants, but these are not used in fitting the Stan model.

This file includes the necessary information to fit the Stan disease natural history model that jointly models viral load, antigen and culture test sensitivity, and time to symptom improvement. It has one input row for each day each study participant was observed. (The variables related to symptom duration are replicated across all rows for a given study participant.) For more information about integers used as indicators in this file, see [`parameters.yml`](`parameters.yml`).

|Column name              |Description                                                                                                                                                                                                                                              |
|:------------------------|:--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|`days_since_symp_onset`  |Number of days relative to day of symptom onset, where day of symptom onset equals zero                                                                                                                                                                  |
|`logVL`                  |Measured logarithm of the viral load ($\log_{10}$ IU / mL); special values include -10 for negative PCR result, 0 for positive PCR result but below limit of quantification, 99 for positive PCR result but above limit of quantification, 100 for not tested |
|`antigen`                |Antigen test result (0 for negative, 1 for positive, 100 for not tested)                                                                                                                                                                                 |
|`culture`                |Antigen test result (0 for negative, 1 for positive, 100 for not tested)                                                                                                                                                                                 |
|`symp_duration`          |Observed symptom improvement day (relative to symptom onset) or last observed day if symptom onservations censored before symptom improvement                                                                                                            |
|`symp_duration_censored` |Whether the symptom duration was censored; i.e., symptom tracking ended before symptom improvement was observed (0 for not censored and 1 for censored)                                                                                                  |
|`lod`                    |Limit of detection ($\log_{10}$ IU / mL)                                                                                                                                                                                                                      |
|`loq_lower`              |Lower limit of quantification ($\log_{10}$ IU / mL)                                                                                                                                                                                                           |
|`loq_upper`              |Upper limit of quantification ($\log_{10}$ IU / mL)                                                                                                                                                                                                           |
|`contiguous_id`          |Integer identifying each study participant                                                                                                                                                                                                               |
|`symp_type_cat`          |Integer identifying symptom category                                                                                                                                                                                                                     |


### `symp_cat_weights.csv`
This file consists of the proportion of study participants in each of the 4 symptom categories. The same information appears in Table S7. These proportions are used as weights to create a synthetic "overall" category. This file is generated by the script [`01_prepare_data_for_stan.R`](scripts/01_prepare_data_for_stan.R).

|Column name     |Description                                                                                                       |
|:---------------|:-----------------------------------------------------------------------------------------------------------------|
|`symp_type_cat` |An integer from 1 – 4 that identifies the symptom category                                                        |
|`wt`            |The weight assigned to each symptom category, calculated as the proportion of study participants in that category |

### `VL_symp_cat_pooled.csv`
This file contains samples from the posterior distribution of the pooled parameters of the hierarchical model that jointly models viral load, antigen and culture test positivity, and symptom improvement time. The model is fit separately for each symptom category. Each row corresponds to 1 draw from a Markov chain. Specifically, 4 chains are fit to each of 4 symptom categories for a total of 16 separate chains, and there are 2,000 post burn-in draws from each chain, for a total of 32,000 rows in this file. Two variables (`symp_type_cat` and `posterior_draw_id`) are needed to uniquely identify a row. This file is generated by the script [`03_extract_parameters_from_stan_fit.R`](scripts/03_extract_parameters_from_stan_fit.R).

|Column name               |Description                                                                                                                                   |Notation (cf. Table S1) |
|:-------------------------|:---------------------------------------------------------------------------------------------------------------------------------------------|:-----------------------|
|`tp_mean`                 |Mean  time of peak viral load, measured in days from symptom onset                                                                            |$t^p$                   |
|`dp_mean`                 |Mean magnitude of peak viral load, measured in log<sub>10</sub> IU / mL                                                                       |$p$                     |
|`wp_mean`                 |Mean proliferation time in days                                                                                                               |$r$                     |
|`wr_mean`                 |Mean clearance time in days                                                                                                                   |$c$                     |
|`sigma`                   |Standard deviation of log<sub>10</sub> viral load measurement error                                                                           |$\sigma_{PCR}$          |
|`tp_std`                  |Standard deviation of time of peak viral load                                                                                                 |$\sigma_{t^p}$          |
|`dp_std`                  |Standard deviation of magnitude of peak log<sub>10</sub> viral load                                                                           |$\sigma_{p}$            |
|`wp_std`                  |Standard deviation of proliferation time                                                                                                      |$\sigma_{r}$            |
|`wr_std`                  |Standard deviation of clearance time                                                                                                          |$\sigma_{c}$            |
|`antigen_50`              |Viral load (log<sub>10</sub> IU / mL) at which antigen positivity is 50%                                                                      |$k_{ant}$               |
|`sigma_antigen`           |Measure of variation (lognormal scale) around threshold where antigen positivity is 50%                                                       |$\sigma_{ant}$          |
|`culture_50`              |Viral load (log<sub>10</sub> IU / mL) at which culture positivity is 50%, if at time 0                                                        |$k_{cul}$               |
|`culture_beta`            |Slope term of linear model where threshold for 50% culture positivity changes with time                                                       |$b_{cul}$               |
|`sigma_culture`           |Measure of variation (lognormal scale) around threshold where culture positivity is 50%                                                       |$\sigma_{cul}$          |
|`posterior_draw_id`       |An integer from 1 – 8000 that identifies the sample from the posterior; re-used across symptom categories                                     |N/A                     |
|`si_shape`                |Shape parameter for Weibull-distributed symptom improvement time distribution                                                                 |$\alpha$                |
|`si_beta_0_exponentiated` |Exponentiated intercept term/baseline scale term for the linear model of the scale parameter of Weibull symptom improvement time distribution |$\exp(\lambda_0)$       |
|`si_beta_wr`              |Measure of association (coefficient in the linear model of Weibull scale) between clearance time and symptom improvement time                 |$\lambda_c$             |
|`symp_type_cat`           |An integer from 1 – 4 that identifies the symptom category                                                                                    |N/A                     |

### `VL_symp_cat_sampled.csv`
This file contains samples from the posterior distribution of pooled and individual-level parameters of the hierarchical model that jointly models viral load, antigen and culture test positivity, and symptom improvement time. The hierarchical model estimates not only pooled parameters that apply to all study participants in a given symptom category but also estimates individual-level “random effect” parameters that define the viral load "triangle" for each study participant. This file is generated by sampling from the `VL_symp_cat_individualized.csv` file (which contains 8,000 posterior samples *for each of the 607 study participants* and is not part of the public release). Specifically, 10,000 samples are randomly selected for each of the 4 symptom categories, yielding a total of 40,000 rows in this file. Some of the columns in this file correspond to individual-level parameters, while others are for pooled parameters that are included here for convenience because they are used downstream in the pipeline (e.g., as part of modeling antigen testing). This file is generated by the script [`04_subsample_extracted_parameters.R`](scripts/04_subsample_extracted_parameters.R).

|Column name     |Description                                                                                                |
|:---------------|:----------------------------------------------------------------------------------------------------------|
|`posterior_id`  |A unique identifier equal to the sample's row number in `VL_symp_cat_individualized.csv`                   |
|`symp_type_cat` |An integer from 1 – 4 that identifies the symptom category                                                 |
|`antigen_50`    |Pooled parameter: Viral load (log<sub>10</sub> IU / mL) at which antigen positivity is 50%                 |
|`sigma_antigen` |Pooled parameter: Measure of variation (lognormal scale) around threshold where antigen positivity is 50%  |
|`culture_50`    |Pooled parameter: Viral load (log<sub>10</sub> IU / mL) at which culture positivity is 50%, if at time 0   |
|`sigma_culture` |Pooled parameter:  Measure of variation (lognormal scale) around threshold where culture positivity is 50% |
|`culture_beta`  |Pooled parameter: Slope term of linear model where threshold for 50% culture positivity changes with time  |
|`tp`            |Modeled individual-level time of peak viral load, measured in days from symptom onset                     |
|`dp`            |Modeled individual-level magnitude of peak viral load, measured in log<sub>10</sub> IU / mL               |
|`wp`            |Modeled individual-level proliferation time in days                                                       |
|`wr`            |Modeled individual-level clearance time in days                                                           |
|`si_time`       |Simulated individual-level observed symptom improvement time in days                                       |

### `intervention_params.csv`
This ancillary file consists of 1 row for each intervention analysis (1 main analysis plus 11 sensitivity analyses). The columns specify the intervention model parameter values used in each analysis and are set in [`parameters.yml`](parameters.yml) (specifically, the `scenario_parameters` and `sens_list` sections). While this file is not used downstream in the pipeline, it is a helpful reference to the combinations of intervention parameter values used in the main and sensitivity analysis and helps interpret the values in the `label` and `analysis` columns in `compare_guidance_df.csv` and `overall_compare_guidance_df.csv`. This file is generated by the script [`05_run_ind_scenarios.R`](scripts/05_run_ind_scenarios.R).

|Column name                                    |Description                                                                                                                                                      |
|:----------------------------------------------|:----------------------------------------------------------------------------------------------------------------------------------------------------------------|
|`infectiousness_integration_dt`                |Time step (days) for integrating under the infectiousness curve                                                                                                  |
|`minimum_days_before_symptom_onset_considered` |Number of days before time = 0 to begin integrating under the infectiousness curve                                                                               |
|`max_si`                                       |Maximum allowed symptom improvement time (days from symptom onset)                                                                                               |
|`infec_threshold`                              |The minimum logVL for someone to be considered "infectious"                                                                                                      |
|`infect_transform`                             |The functional relationship used to relate VL to relative infectiousness                                                                                         |
|`test_count`                                   |How many antigen tests simulated under the previous guidance                                                                                                     |
|`test_interval`                                |The time between antigen tests when there is more than one test when simulating the previous guidance                                                 |
|`iso_lag`                                      |How many days after symptom onset someone starts isolating                                                                                                       |
|`min_iso_days_mild`                            |Minimum number of days someone isolates if they have mild symptoms (previous guidance)                                                                           |
|`min_iso_days_moderate`                        |Minimum number of days someone isolates if they have moderate symptoms (previous guidance)                                                                       |
|`min_total_precaution_days`                    |Minimum number of days someone must take precautions (isolation + post-isolation) (previous guidance)                                                            |
|`min_post_iso_days`                            |Minimum number of days someone must take post-isolation precautions after symptoms improve (updated guidance)                                                    |
|`iso_eff`                                      |Efficacy of isolation in reducing transmission potential                                                                                                         |
|`post_iso_eff`                                 |Efficacy of post-isolation precautions in reducing transmission potential                                                                                        |
|`analysis`                                     |An alphanumeric string that indicates which analysis the output corresponds to (e.g., “main”, “iso_eff”, “iso_eff”); cf. `compare_guidance_df.csv`               |
|`label`                                        |An alphanumeric string that indicates which set of intervention parameters was applied (e.g., “main”, “iso_eff_1”, “iso_eff_0.7”); cf. `compare_guidance_df.csv` |

### `compare_guidance_df.csv`
This is the primary results file with output from the intervention model. The dataset has 480,000 rows, where each row corresponds to a set of parameters sampled from the hierarchical disease natural history model and then applied to each set of intervention parameters. Specifically, the 1 main analysis and 11 sensitivity analyses are all applied to each of 40,000 natural history parameter samples in `VL_symp_cat_sampled.csv`. This file is generated by the script [`05_run_ind_scenarios.R`](scripts/05_run_ind_scenarios.R).

|Column name           |Description                                                                                                                                                        |
|:---------------------|:------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|`label`               |An alphanumeric string that indicates which set of intervention parameters was applied (e.g., “main”, “iso_eff_1”, “iso_eff_0.7”); cf. `intervention_params.csv`   |
|`analysis`            |An alphanumeric string that indicates which analysis the output corresponds to (e.g., “main”, “iso_eff”, “iso_eff”); cf. `intervention_params.csv`                 |
|`posterior_id`        |Unique identifier for the set of disease natural history parameters obtained from hierarchical model and input into the intervention model                         |
|`symp_type_cat`       |An integer from 1 – 4 that identifies the symptom category                                                                                                         |
|`si_time`             |Simulated observed symptom improvement time (days)                                                                                                                 |
|`prop_averted`        |Percentage of transmission potential averted by the updated guidance (compared to not isolating and taking no other precautions)                                   |
|`curr_averted`        |Percentage of transmission potential averted by the previous guidance (compared to not isolating and taking no other precautions)                                  |
|`p_iso_prop`          |Expected time (days) spent in isolation if following the updated guidance                                                                                          |
|`p_iso_curr`          |Expected time (days) spent in isolation if following the previous guidance                                                                                         |
|`p_post_iso_prop`     |Expected time (days) spent in post-isolation precautions if following the updated guidance                                                                         |
|`p_post_iso_curr`     |Expected time (days) spent in post-isolation precautions if following the previous guidance                                                                        |
|`diff_averted`        |Difference in percentage of transmission potential averted by following the updated guidance versus the previous guidance (equals `prop_averted` – `curr_averted`) |
|`diff_iso`            |Difference in expected time (days) in isolation if following the updated guidance versus previous guidance (equals `p_iso_prop` - `p_iso_curr`)                    |
|`diff_post_iso`       |Difference in expected time (days) in post-isolation precautions if following the updated guidance versus previous guidance (equals `p_iso_prop` - `p_iso_curr`)   |
|`intrinsic_I_t`       |Area under the transmission potential curve (in the absence of any interventions)                                                                                  |
|`antigen_detect_prob` |The probability that the infection is detected with antigen assays                                                                                                 |

### `overall_compare_guidance_df.csv`
This file is generated by sampling rows from `compare_guidance_df.csv` in proportion to the symptom category weights stipulated in `symp_cat_weights.csv` to create a synthetic “overall” symptom category. There are 120,000 rows in this file and the columns are the same as in `compare_guidance_df.csv`. Specifically, there are 10,000 samples from the 1 main analysis and 11 sensitivity analyses. This file is generated by the script [`06_create_overall_category.R`](scripts/06_create_overall_category.R).
